Spring Boot Library Management — Project Submission

1. Entity Relationship Design

The application manages two entities: Author and Book.

Entities

Entity Attributes JPA Annotations
Author id (PK), name @Entity, @Id, @GeneratedValue, @OneToMany
Book id (PK), title, genre, author (FK) @Entity, @Id, @GeneratedValue, @ManyToOne, @JoinColumn

Relationship

Author (1) ──────────────── (*) Book One Author can write many Books. Each Book belongs to exactly one Author.

JPA Mapping: ```java // In Author.java @OneToMany(mappedBy = "author", cascade = CascadeType.ALL) private List books = new ArrayList<>();

// In Book.java @ManyToOne @JoinColumn(name = "author_id", nullable = false) private Author author; ```

2. Implementation Details

2.1 Populate Database (DataInitializer)

On application startup, DataInitializer (implements CommandLineRunner) seeds the database with 10 Authors and 10 Books if the author table is empty.

java @Component public class DataInitializer implements CommandLineRunner { @Override public void run(String... args) { if (authorRepository.count() == 0) { for (int i = 1; i <= 10; i++) { Author a = authorRepository.save(new Author("Author " + i)); bookRepository.save(new Book("Book Title " + i, genre, a)); } } } }

Result: 10 rows in AUTHOR table, 10 rows in BOOK table on first run.

2.2 Create Operation

Controller: java @PostMapping("/library/add") public String addBook(@RequestParam String title, @RequestParam String genre, @RequestParam Long authorId, Model model) { try { libraryService.addBook(title, genre, authorId); return "redirect:/library"; } catch (DataIntegrityViolationException e) { model.addAttribute("error", "Data integrity violation: " + e.getMessage()); return "add"; } }

JSP Form (add.jsp): Contains <input> for title and genre, <select> dropdown populated with all authors via JSTL <c:forEach>.

[!NOTE] Exception handling catches DataIntegrityViolationException (e.g., null author FK) and redisplays the form with an error message — the user never sees a blank error page.

2.3 Read Operation

Custom INNER JOIN Query (Repository layer): java // BookRepository.java @Query("SELECT new com.example.library.dto.BookAuthorDTO(b.id, b.title, b.genre, a.name) " + "FROM Book b INNER JOIN b.author a") List<BookAuthorDTO> findBooksWithAuthors();

DTO: java public class BookAuthorDTO { private Long bookId; private String title; private String genre; private String authorName; // constructor + getters }

Controller: java @GetMapping("/library") public String listBooks(Model model) { model.addAttribute("books", libraryService.getBooksWithAuthors()); return "list"; }

JSP (list.jsp): Renders a styled HTML table using JSTL <c:forEach var="book" items="${books}"> with columns: ID, Title, Genre, Author, Action (Edit button).

2.4 Update Operation

Controller (GET — show pre-filled form): java @GetMapping("/library/update/{id}") public String showUpdateForm(@PathVariable Long id, Model model) { Book book = libraryService.getBookById(id); model.addAttribute("book", book); model.addAttribute("authors", libraryService.getAllAuthors()); return "update"; }

Controller (POST — save changes): java @PostMapping("/library/update/{id}") public String updateBook(@PathVariable Long id, @RequestParam String title, @RequestParam String genre, @RequestParam Long authorId, Model model) { libraryService.updateBook(id, title, genre, authorId); return "redirect:/library"; }

JSP (update.jsp): Pre-fills Title and Genre fields using EL (${book.title}, ${book.genre}). Author dropdown pre-selects the current author using ${author.id == book.author.id ? 'selected' : ''}.

5. Challenges Faced & Solutions

Challenge How It Was Solved
JSP not rendering in Spring Boot 3.x Added tomcat-embed-jasper and jakarta.servlet.jsp.jstl (Jakarta EE) dependencies to pom.xml. Old javax.* versions are incompatible with Spring Boot 3.
INNER JOIN returning entity fields, not strings Created BookAuthorDTO with a JPQL constructor expression: SELECT new com.example.library.dto.BookAuthorDTO(...). This lets JPA project query results directly into a DTO.
H2 Dialect warning Removed the explicit spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.H2Dialect from application.properties — Hibernate 6 auto-detects it.
Root URL 404 Added @GetMapping("/") that returns redirect:/library so users landing on the root are forwarded automatically.
